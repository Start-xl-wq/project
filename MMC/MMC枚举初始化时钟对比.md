# 1. 相同点：最高限制均为 400kHz
* 原因：枚举初期（发送 CMD0~CMD3），总线必须工作在开漏输出（Open-Drain）模式以防短路。开漏模式下电平上升沿依靠上拉电阻，非常缓慢。为了保证信号能被正确采样，物理协议严格限制此时钟最高不可超过 400kHz。
# 2. 核心差异：降频重试机制（软件策略）
* SD / SDIO (机械插拔)：存在 400k -> 100k 阶梯降频
    * 物理背景：存在机械触点、易氧化、走线长、信号完整性变数大。
    * 机制：当 400kHz 枚举失败时，驱动层会主动降频（如降至 300k, 200k, 100k）增加电平翻转的时间裕度，这是一种有效的容错/保活手段。
* eMMC (BGA 贴片)：只用 400kHz 刚到底，失败即报错
   * 物理背景：直接贴片焊接在 PCB 上，走线极短且阻抗匹配，信号完整性极高。
    * 机制：如果 eMMC 在 400kHz 下枚举失败，通常意味着发生了致命硬件错误（如虚焊、缺供电、芯片损毁）。此时降频毫无意义。因此，在 Linux 等主流系统中，针对被标记为 non-removable 的 eMMC，驱动不会执行阶梯降频，若有限次重试失败，则直接宣告 eMMC 挂掉。

> 补充冷知识：协议上，SD 卡建议最低初始化频率为 100kHz；而 eMMC 规范（JEDEC）允许初始频率 fODf_{OD}fOD 为
> 0 ~ 400kHz，属于全静态设计。

# 3. IO输出
## 1. 开漏输出 (Open-Drain, OD)
* 内部结构：芯片引脚内部只有一个 N-MOS 管（接地）。
* 形象比喻：就像一个“只能往下拉，不能往上推”的单向开关。
    * 输出 0 时：N-MOS 导通，引脚被死死拉到地 (GND)，输出低电平。
    * 输出 1 时：N-MOS 断开，此时引脚处于悬空状态 (高阻态, Hi-Z)，既不是高也不是低。
* 如何输出高电平？ 必须在外部电路接一个上拉电阻 (Pull-up Resistor) 连到 VCC。当 MOS 管断开时，外部电源通过电阻把引脚电平“慢悠悠”地拉高。
为什么 SD/eMMC 枚举时必须用开漏？ (核心：线与逻辑 & 防烧毁)
* 痛点：在总线刚上电时，主机（Host）不知道总线上挂了 1 个还是 8 个设备（比如早期的 MMC 总线允许多设备并联）。
* 防烧毁：如果用推挽模式，设备 A 强行输出高电平 (VCC)，设备 B 强行输出低电平 (GND)，两极相遇直接短路，芯片当场烧毁。
* 线与逻辑 (Wired-AND)：用了开漏模式后，只要总线上有任何一个设备输出低电平（导通接地的 MOS 管），整条线的电平就是低。只有所有设备都释放引脚（输出 1/高阻态），外部上拉电阻才能把电平拉高。这保证了多设备在总线上同时发声时，绝不会发生电源对地短路。
* 致命缺点（解释了 400kHz 的来历）： 由于上升沿是靠外部上拉电阻给总线寄生电容充电来实现的（RC 充放电）。这个上升过程非常缓慢（斜坡状）。如果时钟频率太快，电平还没来得及爬升到 VCC，下一个时钟就来了，信号直接烂掉。因此，开漏模式的速度物理极限通常在 400kHz 左右。

## 2. 推挽输出 (Push-Pull, PP)
* 内部结构：芯片引脚内部有两个 MOS 管（上面一个 P-MOS 接 VCC，下面一个 N-MOS 接 GND）。
* 形象比喻：就像一个“跷跷板”。
    * 输出 1 时：上面的 P-MOS 导通，下面的关闭，引脚被强力推 (Push) 到 VCC（高电平）。
    * 输出 0 时：下面的 N-MOS 导通，上面的关闭，引脚被强力挽/拉 (Pull) 到 GND（低电平）。
为什么 SD/eMMC 传数据时要切到推挽？ (核心：高速 & 陡峭的边缘)
* 极速开关：推挽模式下，无论是变成高电平还是低电平，都是靠内部晶体管直接导通电源或地。这使得电平跳变极其迅速，上升沿和下降沿像悬崖一样陡峭 (方波完美)。
* 应用场景：一旦 SD/eMMC 结束枚举，主机给它分配了独立地址（RCA），主机确认当前总线上只有这一个设备在跟我说话（一对一通信），就不会有短路风险了。
* 速度飙升：此时协议命令双方立即把引脚切换为推挽输出。因为没有了 RC 充电延迟，时钟频率可以瞬间从 400kHz 飙升到 50MHz (High Speed)、200MHz (HS200/SDR104) 甚至 400MHz (HS400)。
