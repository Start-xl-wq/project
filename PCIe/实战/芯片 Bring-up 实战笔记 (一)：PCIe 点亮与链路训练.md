# 一、 上电前夜的“硬件三板斧”
在怀疑芯片逻辑或软件 Bug 之前，必须先用万用表/示波器确认以下三个物理信号。它们是 PCIe 状态机能够启动的绝对前提。
## 1. Power (电源)
* 主供电 (3.3V)： 模组核心工作电压。
* 辅助供电 (3.3Vaux)： 用于系统休眠时的唤醒功能（如 Wake on LAN）。
* 排查点： 量测金手指上的供电引脚，确认电压是否达标且稳定。
## 2. REFCLK (参考时钟)
* 规格： Host 提供的 100MHz 差分时钟（REFCLK+ / REFCLK-）。
* 核心认知： 无论是 PCIe Gen1 (2.5Gbps) 还是 Gen5 (32Gbps)，板级参考时钟永远都是 100MHz。
* 原理解析： 避免高频时钟在主板上走线带来严重的电磁干扰（EMI）和衰减。Device 拿到纯净的 100MHz 后，内部的 PLL（锁相环） 会按比例倍频，生成实际工作所需的高频时钟（如 Gen1 乘以 25 倍得到 2.5GHz）。
## 3. PERST# (PCIe 复位信号)
* 电平逻辑： 低电平有效（#代表 Active Low）。
* 工作时序： 系统上电 -> 电源稳定 -> REFCLK (100MHz) 稳定 -> Host 将 PERST# 拉高。
* 排查点： 只有 Device 看到 PERST# 被拉高，内部状态机才脱离复位，真正开始 LTSSM（链路训练）。遇到死板子，第一步先看 PERST# 是否被拉高！

# 二、 LTSSM (链路训练状态机) 核心状态深度解析
当 PERST# 拉高后，Host 和 Device 开始通过 LTSSM 建立物理连接，最终目标是进入 L0（正常工作状态）。
## 1. Detect (探测阶段) —— “盲人摸象”
* 作用： 探测插槽上到底有没有插设备。
* 特点： 没有任何 0/1 数字数据交互，纯粹的模拟电路探测。
* 底层原理（RC 充放电）：
    * PCIe 规范要求：设备接收端 (Rx) 必须有 50欧姆对地端接电阻，发送端 (Tx) 走线必须串联 AC 耦合电容。
    * Host Tx 发送一个轻微的共模电压脉冲。
    * 未插卡： 电路开路，没有电容充放电回路，电压瞬间达到峰值（时间常数极短）。Host 判定：空载。
    * 已插卡： Host 的 Tx 通过 AC 电容连到了 Device Rx 的 50欧姆电阻，形成 RC 充放电回路。电压上升变得缓慢。Host 测出较长的时间常数（RC），判定：有设备插入。
* 排查点： AC 耦合电容漏贴/虚焊，会导致永远卡在 Detect。
## 2. Polling (轮询阶段) —— “对齐时钟，提取数据”
* 作用： 保证接收端能够准确无误地从高速串行信号中识别出 0 和 1。
* 痛点： PCIe 没有独立的数据同步时钟线，只有跑满 0/1 的数据线。接收端怎么知道何时采样？
* 核心机制（CDR 与 Lock）：
    * 发测试码： 双方互发特定的训练序列 TS1 和 TS2（包含大量 101010 的跳变沿）。
    * CDR (时钟数据恢复)： 接收端的 CDR 电路看着这些密集的“跳变沿”，不断微调自己的内部采样时钟，直到时钟边沿完美对齐数据眼图的正中心。
    * Bit Lock (比特锁定)： CDR 时钟稳定，能准确切分并识别出每一个 0 和 1。
    * Symbol Lock (符号锁定)： 在 0/1 数据流中找到特殊的定界符（如 8b/10b 编码里的 COM K28.5 字符），从而知道“一个字节”从哪里开始。
* 排查点： 卡在 Polling 多为信号完整性 (SI) 问题。走线太长、干扰大、板材差导致眼图闭合，CDR 无法锁定。
