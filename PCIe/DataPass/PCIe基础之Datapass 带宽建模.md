# (Bandwidth Modeling) —— 性能优化的“高维视角”
【概念】为什么一条 PCIe 3.0 x1 (8 GT/s) 的链路，理论速率是 8Gbps（1000MB/s），但你的 Wi-Fi 跑满最多只能吞吐 700~800MB/s 的净数据（Goodput）？
带宽建模，就是通过数学公式，精确计算出数据在各层流转时的损耗率 (Overhead)。
## 【建模计算示例（自下而上剥洋葱）】
1. PHY 物理层编码损耗：
    * Gen3 使用 128b/130b 编码。
    * 有效率：128/130=98.46%128 / 130 = 98.46\%128/130=98.46%
    * 此时理论上限：1000 MB/s * 98.46% ≈ 984 MB/s
2. DLL 链路层损耗：
    * 每个报文需要加上 SeqNum (2B) 和 LCRC (4B)，加上 Framing 字符。
    * 还要扣除底层偷偷发送的 DLLP（ACK/NAK/UpdateFC）占用的总线时间。
    * 大约损耗 2%~3%。
3. TLP 事务层头损耗（核心杀手）：
    * 如果要发 256B (MPS) 的数据，需要贴上 16B (TLP Header) + 6B (链路层)。
    * 包头效率：256/(256+22)≈92%256 / (256 + 22) ≈ 92\%256/(256+22)≈92%。
    * 如果数据是小包（比如 64B），效率断崖式下跌：64/(64+22)≈74%64 / (64 + 22) ≈ 74\%64/(64+22)≈74%！
4. RX Push / TX Pull 的不对称损耗：
    * RX 方向 (Push)：纯 MWr，发出去就完事，不需要等主板回 TLP 报文。带宽利用率极高。
    * TX 方向 (Pull)：Wi-Fi 发 MRd（占带宽），主板回 CplD（又带了一次包头，再占带宽）。因此相同代数的 PCIe，RX 收包极限通常略高于 TX 发包极限。
# 【一句话总结】 
把协议栈每一层的“包皮”厚度量出来，算出真实果肉占多少百分比，用来寻找系统的性能瓶颈（Bottleneck）。
