# 🟢 L0（正常状态 - 全力输出）
* 是什么：链路全速工作状态，时钟运行，电源满载。
* 进入条件：正常开机枚举完成后，或者从睡眠中被唤醒。
* 如何唤醒：无，正在干活。
---
# 🟡 L0s（浅睡眠 - 单向秒睡）
* 是什么：单向休眠。比如发送端（Tx）没数据发了，Tx 单方面关闭电气信号（进入电气空闲），但接收端（Rx）必须保持开启，等待随时来的数据。主电源、参考时钟（RefClk）全都在运行。
* 进入条件（ASPM 自动触发）：发送端只要闲置一小段极短的时间（微秒甚至纳秒级），硬件状态机（LTSSM）就会自动切断 Tx 电压。操作系统完全不知道。
* 如何唤醒：Tx 突然有数据要发了，立刻发送 FTS（快速训练序列）。对面的 Rx 秒懂，瞬间恢复同步。唤醒极快（通常小于 1 微秒），对性能几乎毫无影响。
---
# 🟠 L1（深睡眠 - 双向共眠）
* 是什么：双向休眠。Tx 和 Rx 两端的收发器全部关闭（双向电气空闲）。内部的锁相环（PLL）可能也会关闭以省电。主电源和参考时钟（RefClk）仍然在运行。
* 进入条件：
    a. 硬件触发（ASPM L1）：总线闲置时间较长，两端硬件互相发包商量：“咱俩都没事干，一起睡吧”，然后进入 L1。
    b. 软件触发（PCI-PM L1）：操作系统要浅休眠了，下发命令把设备设为 D3hot 状态，链路被迫进入 L1。
* 如何唤醒：任何一方有数据要发（或者 OS 强行唤醒）。两边重新打开收发器，通过发送 TS1/TS2 训练序列，经历 LTSSM 的 Recovery（恢复）状态才能回到 L0。唤醒较慢（几微秒到几十微秒），处理不好极易引起掉速或 AER 报错。
---
# 🔴 L1 子状态：L1.1 与 L1.2（极限省电 - 拔掉时钟）
* 是什么：L1 的究极强化版，为了给现代轻薄本续命而生。
    * L1.1：在 L1 的基础上，进一步关掉共模电压。
    * L1.2（最致命）：在 L1.1 的基础上，直接把参考时钟（RefClk）给掐断了！ 芯片内部几乎彻底停摆。
* 进入条件（ASPM 触发）：进入常规 L1 后，设备如果支持 L1.2，会主动把一根叫 CLKREQ# 的物理引脚拉高（取消断言），告诉主板：“我睡死了，把我的时钟关了吧”。
* 如何唤醒： 设备被内部定时器或外界信号吵醒 -> 拉低 CLKREQ# 引脚 -> 主板乖乖把参考时钟重新给上 -> 设备等待时钟稳定、PLL 重新锁定 -> 走 L1 唤醒流程回到 L0。唤醒极慢（几十到上百微秒），很多低端芯片在这里因为时钟恢复失败，直接长眠不醒（死机）。
---
# ⚫ L2（断电守望者 - 幽灵状态）
* 是什么：主电源（Vcc）彻底断电！ 链路彻底物理死亡，所有寄存器配置全部丢失。但是！主板还在提供一个微弱的 Vaux（辅助电源），供设备上的一小块唤醒电路苟延残喘。
* 进入条件（软件触发）：操作系统进入深度休眠（S3 睡眠 / S4 休眠），把设备设为 D3cold 状态。主板随后“啪”一下切断主电源。
* 如何唤醒（经典场景：网卡唤醒电脑）： 设备通过 Vaux 供电的小电路收到外界刺激（比如插上网线、收到网络唤醒包） -> 拉低 WAKE# 边带引脚 -> 主板收到信号，接通全机主电源 -> 主板发出 PERST#（冷复位信号） -> 设备经历完整的开机复位流程 -> OS 重新把之前的配置写回寄存器 -> 回到 L0。
# ☠️ L3（彻底死亡 - 拔掉插头）
* 是什么：主电源（Vcc）没电，辅助电源（Vaux）也没电。0 伏特。
* 进入条件：电脑关机，拔掉电源线。
* 如何唤醒：按电脑主机的实体电源键开机，一切从零开始。
