# 一、 核心概念

> AER（高级错误报告） 是 PCIe
> 体系中的“黑匣子”。它允许底层硬件在发生错误时，将详细的错误类型、发生位置及引发错误的原始数据包记录下来，并通过中断上报给操作系统。作用：将传统的“玄学死机/掉盘”转化为可精确定位的软硬件问题，是排查
> PCIe 链路稳定性、驱动 Bug 和硬件故障的关键机制。

---
# 二、 错误的三个严重等级
PCIe 硬件将错误严格分为三大类，不同类别的处理方式和严重性完全不同：
## 1. Correctable Error（可纠正错误）
* 定义：底层硬件发现了错误，并通过自身的容错机制（如硬件自动重传）成功修复，不需要软件干预。
* 常见类型：
    * Bad TLP / Bad DLLP（数据包 CRC 校验失败）。
    * Receiver Error（接收端物理层错误）。
* 排查方向：如果偶尔发生属于正常；如果高频爆发，通常指向物理层信号质量极差。
    * 常见原因：劣质 PCIe 延长线、金手指氧化、主板布线干扰、或者 ASPM L1 状态唤醒时信号不稳定。
    * 临时解决：在 BIOS 中将 PCIe 降级（如 Gen4 降为 Gen3），或关闭 ASPM 节能。
## 2. Uncorrectable Non-Fatal Error（不可纠正且非致命错误）
* 定义：硬件无法自我修复，导致某个数据包彻底丢失或作废，但整条 PCIe 链路的物理连接仍正常。
* 常见类型：
    * Unsupported Request (UR)（不支持的请求）：请求了不存在的地址或非法的操作。
    * Poisoned TLP（毒化数据包）：数据在传输前已经损坏（如内存 ECC 报错），被打上“有毒”标签继续传输。
* 排查方向：通常指向软件驱动 Bug 或设备逻辑错误。
    * 常见原因：驱动程序写错了内存地址，导致网卡/显卡向一个非法的物理地址发起 DMA 读写。操作系统通常可以通过杀死相关进程或重置该设备来恢复，不需要重启全机。
 ## 3. Uncorrectable Fatal Error（不可纠正且致命错误）
* 定义：链路彻底崩溃，或者数据损坏风险极高，必须立刻停止运行（通常引发 Linux Kernel Panic 或 Windows 蓝屏）。
* 常见类型：
    ◦ Surprise Down（意外掉线）：设备被暴力拔出或突然断电。
    ◦ Completion Timeout (CTO)（完成超时）：CPU 发起读取请求，但设备死机了，超过规定时间（通常为 50ms）未返回数据。
* 排查方向：指向严重的硬件故障或极端的固件 Bug。
    * 常见原因：设备芯片过热保护关机、固件死锁、或者设备进入了 ASPM L1.2 深度休眠后底层时钟未能成功唤醒（死睡不醒）。

# 三、 AER 寄存器组（黑匣子的内部结构）
支持 AER 的 PCIe 设备在其配置空间中有一组扩展寄存器，记录了错误的详细信息：
1. Error Status Register（状态寄存器）：记录当前发生了具体哪种错误（如 UR、CTO 等的标志位）。
2. Error Mask Register（掩码寄存器）：软件可配置硬件忽略某些特定的错误，不进行上报。
3. Severity Register（严重性寄存器）：允许软件将某些非致命错误强行提升为致命错误。
4. Header Log Register（包头日志寄存器 —— 🌟核心调试神器）：
    * 当错误发生时，硬件会将导致错误的那个 TLP（事务层数据包）的前 16 个字节死死锁定在这里。
    * 价值：这 16 字节包含了源设备 ID (Bus:Dev:Func)、目标物理地址、操作类型（读/写）。通过解析它，开发人员可以直接揪出是哪行代码下发了非法的地址。

# 四、 AER 错误上报流程（从硬件到 OS）
1. 错误发生：设备发现错误，将信息记录在自身的 AER 寄存器（包括 Header Log）中。
2. 发送报错报文：该设备在 PCIe 链路上生成一个特殊的 Error Message 报文，向上游发送。
3. Root Port 接收：主板/CPU 端的最上游端口（Root Port）收到报文，记录下是哪个设备的 ID 发来的报错。
4. 触发中断：Root Port 向 CPU 发送系统级中断（MSI/MSI-X 或 NMI）。
5. OS 介入分析：
    ◦ 操作系统的 AER 驱动（如 Linux 的 aer_drv）响应中断。
    ◦ 读取 Root Port 寄存器 -> 找到出事设备的 ID。
    ◦ 读取出事设备的 AER Status 和 Header Log 寄存器。
    ◦ 将解析后的信息打印到内核日志（如 dmesg 中常见的满屏红字 PCIe Bus Error）。
